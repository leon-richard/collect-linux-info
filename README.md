# collect-linux-info

一个使用 C++ 编写的 Linux 系统信息采集程序，旨在高保真地模仿并重现原版 `datacollectforlinux-1_2.5` 采集程序的运行流程、日志格式、错误处理、GBK 编码输出以及数据汇总格式。

---

## 功能特性

1. **高精度日志追踪**：日志输出采用 `HH:MM:SS:Usec` 微秒级高精度时间戳，日志中的中文标点、全宽/半宽冒号与原版程序完全对齐。
2. **多源硬件识别**：
   * **网卡信息**：使用 `getifaddrs()` 动态发现系统中的所有活跃（非环回）IPv4 网卡，查询其 IP 地址，并自动计算紧凑的小写 MAC 地址字符串。
   * **主机与内核**：动态查询主机名，并对内核版本进行智能截断（例如 `5.15.0-xx` 截断为 `5.15.`）。
3. **安全权限自适应与设备链回退**：
   * **普通用户运行**：检测到非 `root` 用户时，输出警告信息到标准错误输出 `stderr`。在获取硬盘序列号时，捕获 `/dev/hda` 和 `/dev/sda` 权限不足或不存在的错误，并安全回退到执行 `lshw -class disk`。
   * **超级用户运行**：成功以 `root` 权限打开设备时，使用 ioctl 查询磁盘信息；当在现代 SATA/SCSI 控制器上调用 `HDIO_GET_IDENTITY` ioctl 返回 `EINVAL` 时，程序可完美捕获并重现相应的系统错误日志，并最终回退获取硬盘序列号。
4. **子进程标准流重定向**：调用 `dmidecode` 或 `lshw` 等系统命令时，其自身的 `stderr` 警告和输出会自然流入终端，而标准输出则会被 C++ 精准捕获并进行格式解析。
5. **原生 GBK 编码输出**：程序内部所有的日志和标签均支持运行时的 GBK 编码转换，能够完美支持通过管道 `| iconv -f GBK -t UTF-8` 输出展示。

## 快速开始

### 编译项目

在项目根目录下，直接使用 `make` 命令进行编译：
```bash
make
```
编译成功后会生成可执行文件 `datacollect`。

### 运行程序

#### 1. 普通用户运行
使用普通用户权限执行时，程序会报告权限警告和部分硬件无法读取的提示：
```bash
./datacollect | iconv -f GBK -t UTF-8
```

#### 2. 超级用户（Root）运行
使用超级用户权限执行时，程序能够读取系统底层的完整硬件序列号：
```bash
sudo ./datacollect | iconv -f GBK -t UTF-8
```

## 输出数据说明

采集完成后，程序会在终端打印硬件信息的文本汇总，并最后输出一行结构化的汇总数据 `CollectData = [...]`。

数据共包含 11 个核心字段，以 `@` 字符作为分隔符：
1. **终端类型**：Linux 系统固定为 `2`。
2. **信息采集时间**：格式为 `YYYY-MM-DD HH:MM:SS`。
3. **私网IP1**：首个活跃网卡的 IPv4 地址。
4. **私网IP2**：第二个活跃网卡的 IPv4 地址（如有）。
5. **网卡MAC地址1**：首个网卡的 lowercase compact MAC 字符串。
6. **网卡MAC地址2**：第二个网卡的 lowercase compact MAC 字符串（如有）。
7. **设备名**：系统主机名（hostname）。
8. **操作系统版本**：截断后的内核主版本号（例如 `5.14.` 或 `5.15.`）。
9. **硬盘序列号**：通过 ioctl 或 `lshw` 获取的物理硬盘序列号。
10. **CPU序列号**：通过 `dmidecode` 获取并经過去空格、大写处理后的 CPU ID。
11. **BIOS序列号**：通过 `dmidecode` 获取的 BIOS 产品序列号。

结构示例：
```text
CollectData = [2@2026-05-21 03:13:21@192.168.33.231@192.168.1.231@bc305bf1d558@bc305bf1d558@ml@5.15.@S0SEN6AC600820@D7060200FFFBEBBF@48JKF3X]
```
